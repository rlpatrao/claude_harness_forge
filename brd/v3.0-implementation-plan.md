# BRD v3.0 — Implementation plan

**Status:** Increment 1a in progress on `claude/implement-claude-agents-aSXLm`.
**Authoritative spec:** [`v3.0.md`](./v3.0.md). This plan owns sequencing and file-level mapping; it does not redefine the spec.
**Cross-session bridge:** [`../harness-progress.txt`](../harness-progress.txt). Live retrofit punch list: [`../feature_list.json`](../feature_list.json).

A future cold-start session can resume from this file alone — every increment lists the exact paths to create, the exact paths to update, and the verification recipe.

-----

## Approach: dogfood the BRD by using v2.0 to build v3.0

The repo *is* the harness. The retrofit uses the forge's own conventions:

- **`feature_list.json` drives sequencing.** Per BRD §3.1 step 6, work the highest-priority failing entry. The retrofit work is itself tracked there.
- **Hooks gate the work.** Once `feature-edit-guard.js` + `e2e-gate.js` + `ralph-loop.js` are registered (increment 1), they enforce themselves on subsequent increments.
- **Existing state files are reused, not replaced.** `state/eval-scores.json` → BRD §4.8 snapshot store. `state/cost-log.json` → BRD §6.1 ledger. `state/iteration-log.md` → BRD §4.8 experiment log. `templates/features-template.json` → BRD §3.2 template seed. `templates/init-sh.template` → BRD §3.1 `init.sh` template. `hooks/cost-tracker.js` + `hooks/token-budget.js` → BRD §3.7 budget footer foundation.

## Resume checklist (for a cold-start session)

1. Read [`v3.0.md`](./v3.0.md) §3, §4, §8 only (the rest is reference).
2. Read [`../harness-progress.txt`](../harness-progress.txt) — last 50 lines tells you what was just done.
3. Read [`../feature_list.json`](../feature_list.json) — find highest-priority `passes: false` entry whose `depends_on` are all satisfied.
4. Cross-reference that entry's `source_section` with the relevant Increment N section above to find the file-level recipe.
5. Implement → verify → flip `passes` (only after a verification artifact lands in `../verification/`) → commit → append progress to `../harness-progress.txt`.
