## Runbook — the happy path (Branch B: full artifact import)

```bash

# 1. Set up BRD + architecture as text files (from the parked prior interview)

cat > /tmp/salary-dashboard-imports/BRD.md <<'BRD'
# Salary Dashboard — BRD (partial, from 2026-05-20 interview)

## Purpose (D1 — Why)
Give job seekers a grounded understanding of US salaries by role and location, using real H1B/OFLC LCA disclosure data. Success = ≥5 correct queries answered per session.

## Scope (D2 — What)
MVP:
1. Browse employer + role + location
2. Wage distribution (percentiles) per role/location cross-section
3. Compare 2 locations for the same role
4. Natural-language Q&A chatbot ("avg SWE base in NYC vs SF FY2024")

Out of scope: employer reviews, contact info, direct application flow.

## Approach (D2.5)
C: DuckDB with materialized views + tool-calling chatbot; LLM proxy backend.
BRD

cat > /tmp/salary-dashboard-imports/architecture.dsl <<'DSL'
workspace "SalaryDashboard" {

            llm_proxy = container "LLM Proxy" "Tool-calling chat orchestrator" "LiteLLM"
```
